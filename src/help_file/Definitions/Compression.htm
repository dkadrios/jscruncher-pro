Compression

The term compression, in regards to JavaScript, refers to the removal of as many portions of script as possible without changing the way the code was meant to run.  Typically this includes the removal of all comments and extraneous whitespace.  Whitespace is defined as tabs, carriage returns, linefeeds and any spaces that serve no purpose.

 

Some other types of compression the engine performs:

String Literal Recombination

Consider the following code:

var vTable =

  '<tr>' +

    '<td>1</td>' +

    '<td>One</td>' +

  '</tr>' +

  '<tr>' +

    '<td>2</td>' +

    '<td>Two</td>' +

  '</tr>';

 

By removing the plus signs and collapsing the quote symbols, JSCruncher reduces this to:

  var vTable='<tr><td>1</td><td>One</td></tr><tr><td>2</td><td>Two</td></tr>';

 

A saving of 28 bytes (characters) with no change in outcome.

 

Superfluous Semicolon Removal

The last line in a code block does not require a semicolon.  JSCruncher will remove any semicolon the precedes an ending curly brace ( } ).  The last line in a file or script tag does not require one either.

 

Obfuscation

JSCruncher Pro's obfuscation engine can add an additional amount of compression to a file, by replacing longer tokens with shorter ones.  If you already use very short (under 5 char) tokens for the majority of your code, obfuscation can actually slightly inflate a file.  On average though, in a healthy sized script, an additional 5 to 10% compression is gained by using obfuscation.
