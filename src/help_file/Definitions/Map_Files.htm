Map Files

Map files contain list of comma separated name-value pairs.

By default all map files are stored in [program dir]\maps, but you can change this location on the settings screen.  A common practice is to use a network share for storing your map files, so that all persons in your organization are using the same maps.

 

Note:  map files are only used when obfuscation is enabled.  This is disabled by default.  To enable it, click on View-> Compression Settings and check the box located on the Obfuscation tab.

Structure of Map Files

A map file consists of a list of name-value pairs.  The left portion is the token to find, the right portion is what to replace it with.  The right side may be omitted, in which case the token is replaced with itself. This action is essential for leaving public references intact.

Here is an example of what the contents of a map file might contain:

 

var1,fef

var2,w4g

myFunc

 

The first two entries are typical of what the obfuscation engine does to non-public references.  You normally wouldn't have entries like this in your map files, but they are shown here as an example.  For these two entries, every instance of var1 that appears in your files will replaced with fef.  Likewise, all instances of var2 will become w4g.

The third entry contains only one value.  This translates to "replace myFunc with itself".  The net effect of this is that myFunc will not be obfuscated.  This is more akin to what your personal map files will contain.  For example, here is a small snippet from the 'ecma.jmapx' file that is included with your installation:

 

Object

offsetHeight

offsetLeft

offsetTop

offsetWidth

onabort

onblur

onchange

onclick

 

Notice that this is just a list of token names.  With this map file in place, every entry in this list will not be obfuscated.

You can read more about this in the section on leaving public references intact.

 

Map File Types

Map files come in two flavors:

 

  • *.jmap - Plain, comma-separated text file

  • *.jmapx - Compressed map file.  These are provided and maintained by Nebiru Software.  You cannot modify these files.

 

From time to time, Nebiru Software makes updated versions of the jmapx files available for download at http://nebiru.com/jscruncherpro.
