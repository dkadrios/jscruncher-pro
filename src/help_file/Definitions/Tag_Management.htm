Tag Management

Tag management refers to the process of renaming the 'src' attributes of <script> includes in your web pages.

When you compress files using JSCruncher Pro, one of the options is to add filename prefixes and suffixes.   For instance, by default the program adds '_c' to all of your compressed filenames.  So if you had a file called 'formValidate.js' and you compressed it, the resulting filename would be 'formValidate_c.js'.  The '_c' helps identify which files have been compressed.

If you then had a webpage that used that script file, such as <script type="text/javascript" src="formValidate.js">, you would want to change that filename to have the '_c' version.  This is what tag management does for you.  You simply select the files you want to have use compressed scripts, and then click 'alter to use compressed scripts'.  The program will then automatically change every script tag it encounters to use the prefixes and suffixes you are using.  There is also a button to 'alter to use normal tags' to reverse the process.

 

Note that Tag Management is only useful if you are using prefixes and suffixes on the compression settings screen.
