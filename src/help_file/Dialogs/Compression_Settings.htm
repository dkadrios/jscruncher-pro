Preamble

  • Enabled

When enabled, the program will prepend the content of compressed files with the text provided.  This can be used to insert a copyright notice or warranty disclaimer.

 

Output

  • Destination

Select the folder that the engine should output the compressed files to.  This should not be the same as the source folder, to avoid accidentally overwriting the original file with the compressed one.

  • Prefix

Value to place in front of each compressed file's name.

  • Suffix

Value to append to each compressed file's name.

 

Obfuscation

  • Enabled

When checked, the compression engine will obfuscate files by replacing tokens with random gibberish - making it harder for someone to reverse engineer your code.

  • Map files to use

Check off which maps files the obfuscation routines should use when looking up tokens.

 

Compression

  • Enabled

Allows you to easily enable or disable the compression routines.  When not checked, the program will not perform any compression routines at all.  Note that this is independent of the obfuscation routines.  You can still obfuscate even if compression is turned off.  This can be useful when debugging your code.

  • Insert missing semicolons

When carriage returns are removed, it is important that your code be well-formed and have each line end with a semicolon.  Normally, the compression engine warns you when it detects that you omitted a semicolon, but you can also have it try to fix the missing one by checking this option.  Note that JSCruncher cannot always detect when a semicolon is missing.  99% of the time it is accurate, but there could be false positives and false negatives, depending on your coding style.  It is recommended that you do not have the program automatically replace missing semi-colons, but you do have that option.

  • Remove Comments

The program will remove both inline and block comments when this option is enabled.

  • Remove whitespace

The engine will remove any of the specified whitespace elements (tabs, carriage returns, etc..).  The engine will also remove any whitespace around the symbols indicated in the symbols grid.

  • Remove extraneous semicolons

If the engine encounters a semicolon that it feels is not necessary, it will remove it.  This can include the last line in a function, the last line in a file or when more that one semicolon appear back-to-back.

  • Combine literal strings

If the engine encounters areas where strings are being added together such as:

var s = 'the first string ' +

             'the second string';

... it will reduce this to:

var s = 'the first string the second string';

 

Symbol grid

The symbols grid defines which symbols will have whitespace trimmed from around them.  There are two columns, one for before the symbol and one for after.  You can add or remove symbols, or check and un-check entries as desired.  Normally the defaults are sufficient, but depending on your coding style, you may need to tweak them.
