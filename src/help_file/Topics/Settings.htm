Settings

The settings screen is divided into 4 main parts:

 

Preamble

Compression

Obfuscation

Miscellaneous

 

Preamble

  • Preamble text

JSCruncher can automatically add a comment header to every file it compresses.  This allows you to maintain copyright information etc., that would otherwise be stripped during the compression operation.  

  • Preamble enabled

Use the edit control on this tab to set your desired preamble.  There is also a checkbox for turning the preamble function on/off.

 

Compression

  • Repository

This the root folder for the compression output.  All files you process will end up under this folder, keeping their original path intact.  For instance, if compress the file c:\my files\tools.js  and your repository is c:\jscrunch\repository\ , the compressed version of tools.js will wind up in c:\jscrunch\repository\c\my tools\.

Your repository can be a local folder, a network share or UNC.

note: while you can specify your root drive as the repository, it is not recommended.  This will cause your original files to be replaced by the compressed versions.  

  • File prefix

If specified, this value will be prepended to all compressed file names.

  • File postfix

If specified, this value will appended to all compressed file names.

  • Report missing semicolons

When enabled, the application will alert you to any missing semicolons.  It is recommended that you leave this turned on.  Missing semicolons can cause your compressed scripts to malfunction.

  • Automatically fix missing semicolons

When enabled, the validation engine will attempt to replace any missing semicolons it encounters.  This will not alter your original files, only the compressed copies.

 

Obfuscation

  • Location of map files

This is the path that JSCruncher will search when looking for obfuscation map files.  This can be a local folder, a network share or UNC.

  • Where to save log files

Where to save the log files if 'Auto save logs' is enabled. This can be a local folder, a network share or UNC.

note: you can also save the log file displayed in the GUI, even if this option is disabled.

  • Obfuscation enabled

Turns obfuscation on/off.

  • Auto save logs

When obfuscation is enabled, JSCruncher will create a log file for each compression session listing the map substitutions it performed.  This is very useful when setting up your initial map files. The log is displayed in the GUI along the bottom of the application, but you can also have it automatically saved to file by enabling this option.

 

Miscellaneous

  • Name of default "user defined" map

In the log that appears at the bottom of the screen after obfuscation, are all the substitutions that JSCruncher made.  If you right-click any of these and select "Never replace", they are placed into what is known as the "user defined" map, which resides in the map folder with the other maps. Here you can change the name of this map file.
