Session Map

After each compression run (when obfuscation is enabled) JSCruncher assembles a special map which contains every token substitution it made.  They are always displayed in a pane along the bottom of the application called the 'session map pane'.  

When you are first crafting map files for your unique project, this pane is invaluable.  With it, you can quickly determine what tokens are being obfuscated.  If you decide any of the tokens need to remain intact, you can either manually add them to your map file(s), or simply right-click them and select "Add to user defined map" which places them in the user-defined map file.

 

The session map is also saved within your log files.  After compressing, click on View --> Log to access it.  The logs also show which files are using which tokens.

 

The session map can also be exported using the button on the pane or by going to File --> Export session map.  This command saves the session map to a .jmap file.

 

note: care should be taken that the public does not get ahold of your map files.  

They could possibly be used to reverse-engineer your obfuscated files.
