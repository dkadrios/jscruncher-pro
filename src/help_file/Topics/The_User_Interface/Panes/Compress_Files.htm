Compress Files

This section allows you to compress JavaScript files.  It presents a tree at the left side of the screen from which you can select a folder, and a file listing at the right from which to select files.  You can double-click any file to invoke the built-in editor.

 

The controls above the file listing include a 'mask filter' control and a checkbox to 'include sub-directories'.   Once all the files you want to compress are showing, select them and press the 'Compress' button [F9] to invoke the compression engine.  By default, it will display the compression settings screen before starting the compression cycle.  

 

You can also press the 'validate files' button [CTRL+F9] to validate the files without compressing.  The validation looks for common errors such as missing semi-colons.  If there are any errors encountered, they will appear in the messages page.   Note that validation occurs automatically during compression, there is no need to run it yourself before each compression.  The option to run it manually is there just in case you wanted to validate the files without actually compressing them.

 

After compressing files, you'll notice several information panes will appear below the file listing.  These include the 'file viewer' which will show the contents of a selected file.  If that will has been compressed, it will also show the compression results side-by-side with the file contents.  There are also tabs for the session map and user defined mapfile panes.  These are only relevant if obfuscation was used during compression.
